# pickmeup

PickMeUp is a personal transport experiment built around a question that sounded fun enough to investigate:

> Where could someone taking the train meet someone driving, so they can continue to the same destination without making either journey unnecessarily awful?

Kuala Lumpur is the test bed. The interesting parts are the data, maps, route choices, waiting, transfers, and all the tradeoffs that appear once two journeys have to meet in the middle.

## Where the project is now

The original 2024 notebook is preserved in [`notebooks/original_exploration.ipynb`](notebooks/original_exploration.ipynb). It contains the first GTFS, NetworkX, OpenRouteService, interchange, and plotting experiments. It also contains shortcuts and assumptions that are useful to study rather than blindly carry forward.

The current package work starts with a GTFS data workbench under `src/pickmeup/gtfs/`. It can:

- download or reopen a Rapid KL rail GTFS ZIP;
- keep every included table available as raw string-valued pandas DataFrames;
- parse GTFS service times, including values after midnight such as `24:15:00`;
- derive route, stop, trip, pattern, and service-time views through normal GTFS relationships;
- report feed quirks without pretending every oddity makes the whole feed unusable;
- check curated interchange stop IDs against a feed snapshot;
- plot route shapes and station locations for quick inspection;
- export a processed snapshot for the browser explorer.

It does **not** choose transit routes or meeting points yet. That comes after the data is understood well enough to make the experiments interesting.

## Static browser explorer

The `docs/` directory contains a build-free HTML, CSS, and JavaScript explorer intended for GitHub Pages. It loads a processed Rapid KL snapshot committed with the repository, so opening the page does not download or parse the official GTFS feed.

The browser dependencies are deliberately small and pinned:

- MapLibre GL JS `5.24.0` from UNPKG for the interactive map;
- the OpenFreeMap Liberty style and OSM-derived vector tiles for detailed Klang Valley roads, neighbourhoods, and place labels;
- Chart.js `4.5.1` from jsDelivr for compact summary charts.

The map is constrained to a generous Klang Valley bounding box and cannot zoom out to a whole-world view. The Rapid KL snapshot remains committed and reproducible, while the background map is loaded on demand from OpenFreeMap. Basemap tiles are not committed or pre-downloaded: a regional tile archive would add repository weight, licensing and refresh work without helping this low-traffic hobby project. Normal browser caching handles repeat views.

There is no npm project, bundler, frontend framework, backend, or API key. The page reads JSON and GeoJSON generated by the Python parser, which remains the source of truth. The basemap is an external best-effort dependency, so roads and labels require internet access even though the rail snapshot itself is cached.

The active snapshot is described by `docs/data/manifest.json`. Its data files live under a checksum-addressed directory such as:

```text
docs/data/snapshots/079a1b5d72cb/
```

Old snapshot URLs are immutable and safe to cache. The small manifest is revalidated by the browser and points to the currently selected snapshot.

To enable the site after this code reaches `main`, open the repository's **Settings → Pages**, choose **Deploy from a branch**, select `main`, and select the `/docs` folder. The expected URL is:

```text
https://noisedoctrine.github.io/pickmeup/
```

### Refreshing the cached feed

The **Refresh GTFS snapshot** workflow is manual only. Run it from the repository's Actions tab when a newer feed is worth inspecting.

The workflow:

1. downloads the current official Rapid KL rail feed;
2. runs the Python exporter;
3. creates a checksum-addressed browser snapshot;
4. opens a reviewable data-refresh pull request when anything changed.

It never writes directly to `main`, and the original GTFS ZIP is not committed. Basemap changes do not use this workflow and should remain ordinary small code edits.

## Setup

PickMeUp requires Python 3.12 or newer.

```bash
python -m venv .venv
source .venv/bin/activate
python -m pip install -e ".[dev]"
```

On Windows, activate the environment with `.venv\Scripts\activate`.

## GTFS workbench

Download the current official Rapid KL rail feed:

```bash
python -m pickmeup.gtfs download data/raw/rapid-rail-kl.zip
```

The ZIP is ignored by Git. A metadata sidecar is written next to it with the source URL, checksum, filenames, row counts, and load time.

Inspect a local feed:

```bash
python -m pickmeup.gtfs summary data/raw/rapid-rail-kl.zip
python -m pickmeup.gtfs quality data/raw/rapid-rail-kl.zip
python -m pickmeup.gtfs curated \
  data/raw/rapid-rail-kl.zip \
  data/curated/interchange_candidate_dispositions.json
python -m pickmeup.gtfs plot \
  data/raw/rapid-rail-kl.zip \
  data/raw/rapid-rail-kl.png
```

Export a browser snapshot from a local feed:

```bash
python -m pickmeup.gtfs export-static \
  data/raw/rapid-rail-kl.zip \
  docs \
  --metadata data/raw/rapid-rail-kl.metadata.json
```

The Python API is intentionally small and notebook-friendly:

```python
from pickmeup.gtfs import GTFSFeed, export_static_snapshot, inspect_feed, plot_network

feed = GTFSFeed.from_zip("data/raw/rapid-rail-kl.zip")
feed.summary()
feed.route_patterns()
inspect_feed(feed).to_frame()
plot_network(feed)
export_static_snapshot(feed, "docs")
```

The 2024 assumptions and the inspected 2026 feed snapshot are compared in [`docs/gtfs_2024_to_2026.md`](docs/gtfs_2024_to_2026.md).

## Repository guide

- `docs/` — static GitHub Pages explorer and cached browser snapshots.
- `src/pickmeup/gtfs/` — current GTFS sourcing, parsing, inspection, plotting, and export code.
- `tests/` — focused tests for objective parsing, relationships, and deterministic snapshot export.
- `notebooks/original_exploration.ipynb` — the preserved 2024 prototype.
- `notebooks/legacy/interchanges.pickle` — the original interchange DataFrame retained as evidence.
- `_interchanges_csv_for_inspection_only.csv` — a readable export of that legacy data.
- `data/curated/interchange_candidate_dispositions.json` — reviewed interchange decisions that later experiments can consume.
- `docs/interchange_candidate_validation.md` — notes on how those candidate decisions were investigated.

## API key

OpenRouteService is not needed for GTFS parsing or the static explorer. Later road-routing experiments will use the `OPENROUTESERVICE` environment variable shown in `.env.example`.

Do not commit real credentials.
